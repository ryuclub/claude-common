# 通用开发指南

本目录包含**所有项目通用**的开发规范和指南。这些指南不包含语言或项目特定的细节，可直接复用到其他项目。

---

## 📋 导航

### 第 1 层：项目启动必读

1. **[开发工作流](./01-workflow.md)** — 7 步工作流
   - 需求 → 设计 → 代码 → 测试 → 审查 → 部署 → 验收
   - 适用：所有项目的所有任务

2. **[设计文档规范](./02-design-document.md)** — 如何编写设计文档
   - 文档结构、内容要求、审查标准
   - 适用：工作流 Step 2（设计阶段）

### 第 2 层：开发过程中参考

3. **[分支管理规则](./03-branch-management.md)** — Git 分支规范
   - 分支命名、提交规范、PR 流程
   - 适用：所有 Git 操作

4. **[编码原则](./04-coding-principles.md)** — 语言无关的编码原则
   - 命名规则、代码组织、可读性、错误处理
   - 适用：所有代码编辑任务
   - 注：具体实现由项目的 `.claude-local/standards/` 细化

5. **[代码审查清单](./05-review-checklist.md)** — 代码审查标准
   - 审查要点、质量检查、安全审查
   - 适用：所有代码审查任务

## ✅ 特点

- **通用**：不包含语言特定的细节（Go/Python/Node 都适用）
- **可复用**：可直接复制到任何项目
- **清晰**：编号便于导航和参考
- **简洁**：只包含必要的原则和规范，不重复
